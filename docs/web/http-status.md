# HTTPステータスコード

## 概要
HTTPレスポンスの状態を3桁の数値で表現する仕組み。

## 分類
- 1xx: 情報
- 2xx: 成功
- 3xx: リダイレクト
- 4xx: クライアントエラー
- 5xx: サーバーエラー

## 主要ステータスコード
- 200 OK: リクエスト成功
- 201 Created: リソース作成成功
- 204 No Content: 成功（レスポンスなし）

- 301 Moved Permanently: 恒久的リダイレクト
- 302 Found: 一時的リダイレクト

- 400 Bad Request: リクエスト不正
- 401 Unauthorized: 認証されていない
- 403 Forbidden: 権限なし
- 404 Not Found: リソースが存在しない

- 500 Internal Server Error: サーバー内部エラー
- 502 Bad Gateway: 上流サーバーエラー
- 503 Service Unavailable: サーバー利用不可

## 重要な違い
### 401 vs 403
- 401: 認証していない
- 403: 認証済みだが権限がない

### 301 vs 302
- 301: 恒久的（SEO影響あり）
- 302: 一時的

## 実務での考え方
- API設計ではステータスコードを適切に返すことが重要
- フロントエンドはステータスコードに応じて処理を分岐する
